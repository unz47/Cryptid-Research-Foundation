## 🎯 コスト最適化戦略

### 即効性の高い施策

#### 1. 画像最適化（月$30-60削減）

**問題**: 画像データ転送が全体の60-80%

**解決策**:
```typescript
// next.config.js
module.exports = {
  images: {
    formats: ['image/avif', 'image/webp'],
    deviceSizes: [640, 750, 828, 1080, 1200, 1920],
    minimumCacheTTL: 31536000, // 1年
  },
};
```

**効果**:
- AVIF: 原画比50-70%削減
- WebP: 原画比25-35%削減
- データ転送量: 40-60%削減
- **月$30-60のコスト削減**

#### 2. ISR導入（月$10-20削減）

**問題**: すべてのリクエストでLambda実行

**解決策**:
```typescript
// app/creatures/[slug]/page.tsx
export const revalidate = 3600; // 1時間

export default async function Page({ params }) {
  const data = await fetch(`/api/creatures/${params.slug}`, {
    next: { revalidate: 3600 }
  });
  return <div>{/* ... */}</div>;
}
```

**効果**:
- Lambda実行: 80%削減
- CloudFrontキャッシュヒット率: 90%以上
- **月$10-20のコスト削減**

#### 3. CloudFrontキャッシュ最適化（月$15-30削減）

**設定**:
```typescript
// CloudFrontキャッシュポリシー
{
  "/_next/static/*": {
    "TTL": 31536000, // 1年
    "compress": true
  },
  "/api/*": {
    "TTL": 3600, // 1時間
    "s-maxage": 86400 // 1日
  }
}
```

**効果**:
- オリジンリクエスト: 70-90%削減
- **月$15-30のコスト削減**

## 🔄 フェーズ別推奨構成

### Phase 1: 初期（1-3ヶ月）

**推奨**: **Lambda/SST**
**理由**: Free Tier内、最低コスト
**月額**: $0-6

**設定**:
- Lambda: 512MB（十分）
- Warmer: 5個
- ISR: 有効

### Phase 2: 成長（4-9ヶ月）

**推奨**: **Lambda/SST** または **Amplify**
**理由**: コスパ最高
**月額**: $14-18

**最適化ポイント**:
- 画像最適化実施
- ISR活用
- CloudFrontキャッシュ強化

### Phase 3: 確立（10-12ヶ月）

**推奨**: **Lambda/SST + Savings Plans**
**理由**: 継続的に低コスト
**月額**: $60（最適化後$29）

**検討事項**:
- Compute Savings Plans（17%割引）
- Lambda Reserved Concurrency
- CloudFront Savings Bundle

### Phase 4: スケール（12ヶ月以降）

**選択肢**:

**A. Lambda/SST継続**（推奨）
- 月10万PV以下: $29-60
- 完全自動スケール

**B. Fargate移行**（高トラフィック）
- 月10万PV以上: $200-500
- 安定パフォーマンス

### Cost Anomaly Detection

自動的にコスト異常を検知してアラート送信。

**設定**:
```
AWS Console → Cost Management → Cost Anomaly Detection
→ "Create monitor"
→ サービス: Lambda, CloudFront, S3選択
→ 通知: SNS or Email
```
